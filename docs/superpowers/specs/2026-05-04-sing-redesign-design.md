# Sing site redesign — design spec

**Date:** 2026-05-04
**Status:** Draft, pending review
**Scope:** Frontend redesign of the Baylor Sing data site, focused on the 2022–2025 cycle

## Goal

Replace the current generic data site with a theatrical, narrative-driven experience that uses the now-real color palette data alongside Spotify audio features to tell the story of Sing across four years (2022–2025). The reader should walk away feeling they've attended the show, while also having seen real analytical findings about color, sound, and what wins.

## Scope

**In scope.**
- A new site at `site_v2/`, built alongside the existing `site/` and swapped when ready.
- Seven narrative scenes (Curtain Rise → Cast → Sound → Color → Verdict → Behind the Curtain → Encore).
- Vanilla JavaScript, hand-rolled SVG charts, `scrollama` for scroll-pinning.
- Data window: 67 acts from 2022–2025, drawn from the existing data pipeline.

**Out of scope.**
- Reworking pre-2022 acts (the existing site remains the historical record for those).
- The image collection / palette extraction pipeline itself (already rebuilt).
- A backend; this stays a static site.
- Mobile-first design (desktop-first; mobile is a degraded but functional fallback).

## Architecture

### Scene structure

Seven scenes, scrolled top-to-bottom on a single page. Theatrical scenes (I, V, VI, VII) use full-screen moments with red-velvet motifs. Editorial scenes (II, III, IV) are scrollytelling — chart pins to viewport while explanatory copy scrolls past.

| # | Scene | Mood | Primary content |
|---|-------|------|-----------------|
| I | Curtain Rise | Theatrical | Animated gold "Sing" logo with curtain reveal, framing line, scroll cue |
| II | The Cast | Editorial | 67-tile grid of all acts as palette mini-strips; sortable; tile click scrolls reader to that act's location in later scenes |
| III | The Sound | Editorial | Audio Map (valence × energy scatter) → Top 20 artists → Song age → Genre proportions |
| IV | The Color | Editorial | Years in Color (palette stacks) → Color Wheel → Saturation × Brightness scatter → Extremes callouts |
| V | The Verdict | Theatrical | Pigskin reveal, winner-vs-participant comparison, four 1st-place spotlight cards |
| VI | Behind the Curtain | Theatrical | Two short paragraphs explaining how song data and color data were extracted |
| VII | Encore / Fin | Theatrical | Three or four big-number findings, "Fin." closer, credits/sources |

### Data pipeline

Data is built once at site-build time, not loaded ad hoc.

1. `scripts/06_build_site.py` (rewrite) reads:
    - `data/sing_enriched.parquet` (year, group, theme, placement, songs, audio features)
    - `data/color_palettes.csv` (palette_hex, proportions, image_source, n_images)
2. Script filters to 2022–2025, joins palettes onto acts, computes derived fields per act:
    - `dominant`: most-vivid color (`max(S × V × (1 + 0.3 × proportion))` over top 6 colors)
    - `avg_hue`, `avg_sat`, `avg_val`: weighted averages over palette colors with `V ≥ 0.15`
    - `palette_source`: `'youtube'` (real) or `'bing'` (placeholder; Scene II shows "no video available" badge)
3. Output: `site_v2/data/acts.json` — one array of 67 act objects.

The parquet is regenerated by re-running the existing pipeline (`scripts/04_merge_dataset.py`) before this script. That step is out of scope for this design but already exists.

## Visual system

### Typography
- **Display / titles:** `Playfair Display`, 700–900 weight, italic for theatrical moments. The script-style "Sing" wordmark in Scene I is custom (SVG path, gold gradient).
- **Editorial body:** `Inter`, 400/500. Body sets ~16px / 1.6 line-height.
- **Labels / small print:** `Inter`, 500, uppercase, 1–2px letter-spacing.
- **Numbers in callouts:** `Playfair Display`, italic, very large (e.g., 88px) with gold gradient fill.

### Color tokens
| Token | Hex | Usage |
|-------|-----|-------|
| `--velvet` | `#6b1117` | Curtain red |
| `--velvet-deep` | `#2a0608` | Deep velvet shadow |

| `--text-tertiary` | `#5a4a3a` | Muted, axis tick numbers |

### Motion
- **Curtain reveal (Scene I):** CSS transform — left/right curtains animate from closed to open over 1.2s at page load.
- **Logo fade-in:** Gold "Sing" SVG fills via stroke-then-fill animation, 0.8s, easing on the end.
- **Scroll-driven:** scrollama steps; each chart has `enter` / `exit` callbacks that highlight the relevant data point or annotation.
- **Hover:** All chart dots scale up 1.2× on hover with a 0.15s ease.

## Per-scene specs

### Scene I — Curtain Rise
Full-viewport hero. Red gradient background. Two red-velvet curtain panels (CSS gradients with vertical pleat striping) start closed and animate open. Center: gold "Sing" wordmark on a velvet stage. Subtitle line: "Four years. Sixty-seven acts. The data behind the show." Scroll cue (down arrow, fading in/out) at bottom.

### Scene II — The Cast
Header: "The Cast." Sub: "Sixty-seven acts, 2022–2025."
Below: a year-grouped grid of palette tiles (4 year bands: 2022, 2023, 2024, 2025). Each tile shows the act's full 10-color palette as a horizontal strip on top, then the dominant color square + group/theme/placement/source-tag in the body. Click a tile → smooth-scroll to that act's audio map dot in Scene III, with a temporary highlight ring.

Acts with `palette_source: 'bing'` show a small `no video` badge and use a desaturated grey placeholder palette instead of the bing palette (those palettes are unreliable and shouldn't shape the reader's mental model).

Sortable by year (default), Pigskin status, or vivid pick hue. Sort change re-orders tiles in place with a 0.3s transition.

### Scene III — The Sound
Four sub-sections, each scroll-pinned.

1. **Audio Map** (the chart we prototyped). Valence × energy scatter, dot color = vivid pick, gold rings for medals, bronze rings for Pigskin. Tooltip shows full palette + theme. **Includes a small explainer block** describing what valence (musical positivity) and energy (intensity) measure. Bing-fallback acts use a neutral grey dot fill (consistent with their treatment in Scene II) so unreliable color data doesn't shape the chart.
2. **Top 20 Artists.** Horizontal bar chart, gold bars, ranked by # of acts that covered each artist's songs. Year labels under each bar's name.
3. **Song Age.** Histogram of (Sing year − song release year) across all songs in scope. Annotations call out interesting peaks ("most acts use songs ~5 years old at performance time").
4. **Genre Proportions.** Stacked area / stream chart showing the genre mix per year.

Each chart has a sticky title + a paragraph of editorial copy that scrolls past; the chart highlights the relevant subset as the reader passes (e.g., as the Song Age paragraph mentions Disney covers, the bar bucket for >50-year-old songs lights up).

### Scene IV — The Color
Four sub-sections, mirroring the four chart prototypes.

1. **Years in Color.** Four stacked horizontal palette strips (one per year). Within each strip, acts are laid side-by-side with their full palette stacked vertically inside their slot. Hover an act to enlarge.
2. **Color Wheel.** Polar scatter: angle = dominant hue, radius = saturation. Dot fill = the act's actual dominant color. Reveals warm/cool clustering.
3. **Saturation × Brightness Scatter.** The visual analog of the Audio Map. Dots positioned by avg_sat × avg_val, colored by vivid pick.
4. **Extremes.** Four cards: most vivid, most muted, brightest, darkest. Each shows the dominant swatch + group + theme.

### Scene V — The Verdict
Heavy theatrical return. Three sub-sections.

1. **The Pigskin reveal.** Big gold star/medal motif, headline like "Thirty-three acts made Pigskin. Twelve took home medals. Four were crowned." A grid of all top-8/Pigskin acts across the four years, with medal acts visually elevated (gold double ring, label) and the four 1st-place winners largest. Exact counts: 12 medal acts (1st × 4 + 2nd × 4 + 3rd × 4) + 21 additional Pigskin = 33 top placements over 4 years.
2. **Winner vs participant comparison.** Two side-by-side panels: average palette + average audio fingerprint (radar of valence/energy/danceability/tempo) for Pigskin acts vs Participated acts. Reveals what (if anything) separates them.
3. **First-place spotlights.** Four cards, one per year (2022–2025): the 1st-place act, with name, theme, palette, songs, and a quote-style framing.

### Scene VI — Behind the Curtain
Single screen, two short paragraphs side by side or stacked, framed by a simple "stage manager" header. Each ~80–120 words.

- **Songs:** "We named the songs in each act, then enriched them through the Spotify API to get audio features (valence, energy, danceability, tempo, acousticness, loudness) and genre tags. Where Spotify had multiple matches, we picked the most popular. Audio averages per act are means across the act's songs."
- **Color:** "We pulled YouTube videos for each act, sampled eight evenly-spaced frames per video, and ran K-means clustering (K=10) on the pooled pixels. Near-black background and very desaturated pixels were filtered out so costume and set colors would dominate. The 'dominant' color for each act is the most vivid one — highest saturation × brightness — among the top six by share."

### Scene VII — Encore / Fin
Three or four big-number callouts (Playfair italic, gold gradient): findings like "The average Sing song scores 0.74 valence — happier than 90% of pop", "Greens dominate winners three years running", "The most-covered artist of the era is __".

Then a soft fade to a velvet background with a gold "Fin." in script-style, beneath which a small footer block: data sources (Spotify, baylor-sing-all-acts-final.csv, Baylor Sing Archive YouTube channel), GitHub link, methodology link.

## Component contracts

Each chart module in `js/charts/` exports a single function:

```js
export function render(rootElement, acts, options = {}) {
  // builds SVG / DOM into rootElement
  // returns { highlight(filterFn), reset() } for scrollama callbacks
}
```

The Cast tile click → audio map highlight is handled in `main.js` by holding a registry of rendered chart instances and calling `audioMap.highlight(d => d.year === yr && d.group === grp)`.

## Risks and mitigations

- **Bing-fallback palettes contaminate downstream charts.** Mitigation: charts that aggregate (color wheel, sat-val scatter, extremes) only include `palette_source === 'youtube'` acts. The Cast scene is the only place bing-fallback acts appear, and they're tagged.
- **Data refresh.** New Spotify enrichment runs are out of scope for this redesign but if they're done, the build script picks up automatically (it just reads the parquet).
- **Scrollama jank on first paint.** Tested in current site, works fine. Use the same vendored copy.

## Acceptance criteria

- All seven scenes render with real data, no placeholder copy left in code.
- Cast grid shows all 67 acts with correct palettes; bing-fallback acts are visually distinct.
- Audio map shows 65 plotted dots (the 2 with NaN audio features get a "missing" footnote).
- Color wheel and sat-val scatter only include the 51 YT-derived acts.
- Verdict scene shows correct counts (1st × 4, 2nd × 4, 3rd × 4, additional Pigskin × 21 = 33 top placements over the 4 years; 34 Participated; 67 total) — verify against `baylor-sing-all-acts-final.csv`.
- The site runs from a static file server (e.g., `python3 -m http.server`) without additional tooling.

## Implementation strategy

This will be a multi-phase implementation plan written separately. High-level phasing:

1. Build script + data shape (`acts.json` with everything baked in).
2. Layout shell (HTML structure, CSS tokens, scene scaffolding, scrollama wiring).
3. Editorial scenes (II, III, IV) — Cast grid, then chart-by-chart.
4. Theatrical scenes (I, V, VI, VII) — curtain animation, verdict reveal, methodology, encore.
5. Polish pass — scroll-driven highlights, hover states, mobile fallback.
6. Cutover — rename `site_v2/` → `site/`, archive previous as `site_legacy/`.
